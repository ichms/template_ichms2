# SCALE-01 Scaling and Adoption

- Priority: P2
- 기본 Enforcement: Guide
- 적용 범위: 기존 기능 리팩토링 + 신규 도입 계획

## 분리 트리거

라인 수만 단일 지표로 사용하지 않고 3축으로 판단한다.

- 축 A (크기): 단일 파일 350+/500+/600+
- 축 B (복잡도): 조건 분기 8개+
- 축 C (렌더 영향): 불필요 재렌더 hotspot 확인(Profiler)

## 단계별 운영 기준

- `Warning`: A/B/C 중 1개 충족
- `분리 계획 필수`: 2개 충족
- `Hard Gate(예외)`: 3개 충족 또는 600+와 hotspot 동시 충족

`Hard Gate(예외)`는 Priority가 아니라 Enforcement 승격 케이스다. EX-01 템플릿으로 기록한다.

## 분리 원칙

- 처음부터 과분할하지 않는다.
- 복잡도 지표를 만족할 때만 분리한다.
- 분리 후에도 파일 책임 1개 원칙을 유지한다.

## 도입 로드맵

1. Phase 1: 규칙 합의 + 템플릿 + 리뷰 체크리스트 도입
2. Phase 2: 신규 기능부터 적용
3. Phase 3: 장애/변경 빈도 높은 도메인부터 리팩토링
4. Phase 4: 생성기/린트/PR 템플릿 자동화

## 리뷰 체크 (Yes/No)

- Scale Check가 PR에 기록되었는가?
- 트리거 충족 개수에 맞는 조치(경고/분리계획/차단)가 적용되었는가?
- 차단 케이스에 예외 승인 정보가 포함되었는가?
