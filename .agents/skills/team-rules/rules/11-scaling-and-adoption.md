# SCALE-01 Scaling and Adoption

- Priority: P2
- 적용 범위: 기존 기능 리팩토링 + 신규 도입 계획

## 분리 트리거 (MUST)

- 단일 파일이 350줄을 초과하면 경고를 남긴다.
- 단일 파일이 500줄을 초과하면 분리 계획을 반드시 남긴다.
- 단일 파일이 600줄을 초과하면 예외 승인 없이는 머지하지 않는다.
- 분리 계획/예외 기록에는 분리 대상, 예상 분리 파일, 담당자, 목표 일정을 포함한다.
- 350/500/600 단계 기록은 PR 템플릿의 Scale Check 섹션에 필수로 남긴다.
- 기록이 없는 경우 규칙 미준수로 판단한다.

## 분리 원칙

- 처음부터 과분할하지 않는다.
- 복잡도 지표를 만족할 때만 분리한다.
- 분리 후에도 파일 책임 1개 원칙을 유지한다.

## 도입 로드맵

1. Phase 1: 규칙 합의 + 템플릿 + 리뷰 체크리스트 도입
2. Phase 2: 신규 기능부터 강제
3. Phase 3: 장애/변경 빈도 높은 도메인부터 리팩토링
4. Phase 4: 생성기/린트/PR 템플릿 자동화

## 리뷰 체크 (Yes/No)

- 350줄 초과 파일에 경고가 기록되었는가?
- 500줄 초과 파일에 분리 계획이 기록되었는가?
- 600줄 초과 파일에 예외 승인 또는 머지 보류가 적용되었는가?
- 도입 단계가 팀 역량과 일정에 맞는가?

## 자동 검증

- 린트: 제한적
- 리뷰 수동: 필요
